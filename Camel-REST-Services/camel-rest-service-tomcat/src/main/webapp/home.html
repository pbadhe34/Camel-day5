Apache Camel Web using Servlet REST & Web Container

This shows how to use Servlet REST to define REST endpoints in Camel routes using the Rest DSL
For Java DSL the routes are defined in Java code, in the org.app.RestRouteBuilder class.

There is a provider REST service that supports the following operations

  • GET /rest/provider/{id} - to view a provider with the given id
  • GET /rest/provider/ - to view all providers
  • PUT /provider - to update/create an provider
The view operations are HTTP GET, and update is using HTTP PUT. From a web browser you can access the first two services using the following links
  • provider/123 - to view the provider with id 123
  • provider/ - to list all providers
From the command shell you can use curl to access the service as shown below: 
    curl -X GET -H "Accept: application/json" http://localhost:8080/camel-rest-web/rest/provider/123
    curl -X GET -H "Accept: application/json" http://localhost:8080/camel-rest-web/rest/provider/findAll
    curl -X PUT -d "{ \"id\": 999, \"name\": \"Dr. Who\"}" -H "Accept: application/json" http://localhost:8080/camel-rest-web/rest/provider
Notice when trying to get provider with id 789 then there is a built-in delay of 0.5-2 seconds, to simulate some slowness in the service.

This offers an API browser using Swagger which is accessible from the following link

  • api-docs - overview of the apis from the REST services
  • api-docs/provider - api of the provider REST service

To use the swagger ui, follow this link. Replace the URL value with this one to access the REST local resources (http://localhost:8080/api-docs or (http://localhost:8080/camel-rest-web/api-docs when deployed in Apache Tomcat)
